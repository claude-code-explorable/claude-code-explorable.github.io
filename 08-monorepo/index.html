Claude Code in a monorepo

One repository, many teams. A single root CLAUDE.md either balloons to cover every subsystem or stays uselessly generic — so you layer instructions by package and stop loading the whole tree. And because teammates share the repo, what the harness allows is not one file but four scopes merged: the org's managed rules, the team's committed settings, your local overrides, and your user defaults. Turn two dials — where you start Claude and who you are — and watch each teammate's session assemble.

How to read this. The start dial decides what loads: the root CLAUDE.md always loads, your starting directory's loads with it, and everything else only loads on demand — or never, if you started deep. Start in packages/api and the whole web subtree falls out of scope for free. The who dial swaps your personal .claude/settings.local.json — an API dev excludes other teams' packages and reaches ../shared; the platform team authors the managed layer no one can override. Below the tree, the four scopes merge into a single gate where deny always wins and the org's rules are locked. Watch the same request come out allowed for one teammate and blocked for another — from the same repo.
